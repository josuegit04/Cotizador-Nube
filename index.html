Configuración

Clientes Base (Valle):iEl tráfico mínimo constante diario (ej: la madrugada). Define el origen vertical de la curva.
Clientes Peak 1:iLa cantidad más alta de usuarios concurrentes esperada en el primer pico de tráfico del día.
Hora del Peak 1 (0-24):iLa hora exacta donde ocurrirá el primer pico. Matemáticamente sitúa el primer vértice.
Clientes Peak 2:iTráfico máximo esperado para el segundo momento de alta demanda.
Hora del Peak 2 (0-24):iHora del segundo pico. Python enlazará ambas curvas de forma continua.
Región Cloud:iLa ubicación física del Data Center. Sudamérica suele ser más costosa debido a la latencia y enlaces locales.
Base de Datos (GB):iEspacio en disco para el motor relacional. Se tarifica mensualmente por GB provisto.
Ficheros en Buckets:iAlmacenamiento de objetos masivos independientes (ej: imágenes de usuarios, PDFs).
Balanceador de Carga:iDistribuye las peticiones entre servidores cuando la pendiente (tasa de cambio) se acelera súbitamente.
